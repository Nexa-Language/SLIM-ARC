    \item \textbf{tradeoff 对交互式推理有利}：decode +262\% 的收益远大于 prefill --57\% 的代价。
\end{enumerate}

\subsection{数据波动分析}

在 80B 大模型的实验中，我们观察到显著的数据波动。同一配置的多次测量结果可能相差 2--3 倍。主要原因包括：

\begin{enumerate}
    \item \textbf{Page cache 状态差异}：80B 模型（45\,GB）远超 32\,GB 物理内存，每次推理都需要从 SSD 加载部分页面。page cache 命中率取决于系统其他进程的活动，导致每次运行的 I/O 延迟不同。
    \item \textbf{cgroup 内存回收干扰}：当 cgroup 接近 memory.max 时，内核会频繁回收页面，引入不可预测的延迟。
    \item \textbf{NVMe SSD 的 GC 机制}：SSD 的垃圾回收行为会影响随机读取的延迟。
\end{enumerate}

为缓解数据波动的影响，所有核心实验均采用冷启动（drop\_caches）+ 多次测量的方法。报告中呈现的数据为多次测量的代表值，完整原始日志保存于 \texttt{logs/ablation/} 目录。

\subsection{StreamingLLM KV Eviction}

受综述 Sec 4.3 "combine KV compression with eviction policies (e.g., window + sink tokens)" 启发，我们实现了 StreamingLLM 式的 KV Cache 驱逐机制 \cite{streamingllm2024}。同时参考了 SolidAttention \cite{solidattention2026} 的 block-wise attention sparsity 思路。

\subsubsection{机制}

在每次 decode step 后，检查 KV 序列长度。若超过阈值 $\text{sink} + \text{window}$，则驱逐区间 $[\text{sink}, \text{seq\_len} - \text{window})$ 内的 token，仅保留：
\begin{itemize}
    \item \textbf{Attention sinks}（前 $N=4$ 个 token）：稳定 softmax 分母，避免 attention 分布退化
    \item \textbf{Sliding window}（最近 $W$ 个 token）：保持近期上下文
\end{itemize}

这使 KV Cache 内存从 $O(\text{context\_length})$ 降至 $O(N + W)$，在长上下文场景下显著减少内存占用。驱逐通过 llama.cpp 的 \texttt{llama\_memory\_seq\_rm} 接口实现，无需修改 attention kernel。

\subsection{Speculative Decoding 负面结果}

受综述 Sec 4.5 "speculative decoding [282] align computation depth with input complexity; CE-LSLM achieve up to 6$\times$ speedup" 启发，我们测试了 llama.cpp 内置的 speculative decoding 在 80B MoE 模型上的效果。

\subsubsection{实验配置}

\begin{itemize}
    \item 模型：Qwen3-Next-80B-A3B IQ4\_XS（同模型做 draft + target，因 Qwen3-Next 无 small 版本）
    \item 模式：\texttt{ngram-simple}（n=8, m=4，不需要额外 draft model）
    \item 环境：32\,GB, 8 threads, KV q4\_0
    \item 任务：48 token 生成
\end{itemize}

\subsubsection{结果}

\textbf{分析}：Speculative decoding 在 80B MoE 上呈现净负收益（-53.2\%），原因包括：

\begin{enumerate}
    \item \textbf{无 small draft model}：Qwen3-Next 系列没有 0.5B--1B 的小版本，用 80B 自身做 draft 无法获得小模型的计算加速。
    \item \textbf{MoE router 高熵}：MoE 模型的 expert routing 增加了 token 序列的不确定性，n-gram 预测的 accept rate 仅 33.3\%。
    \item \textbf{验证开销}：每个 draft token 都需要 80B 的完整 forward pass 验证，当 accept rate 低时，验证开销远超投机收益。
\end{enumerate}

这一结果与 unsloth 对 Qwen3.6-35B-A3B 的测试一致\footnote{\url{https://huggingface.co/unsloth/Qwen3.6-35B-A3B-GGUF/discussions/14}}，证实了 speculative decoding 对 MoE 大模型的局限性。我们将其作为负面结果诚实记录，避免后续工作重复探索此方向。
