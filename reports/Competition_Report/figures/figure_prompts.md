# SLIM-ARC 报告示意图设计描述（NeurIPS/AAAI 标准）

本文件为每个示意图提供自然语言设计描述，供后续用 draw.io / TikZ / Figma / 手绘实现。
设计原则：**少文字多图、不规整普通、自包含自解释**。描述不限定具体布局，只传达视觉意图。

## 图 1：SLIM-ARC 三层整体架构（fig:architecture）

### 设计意图
打破常见的"三条平行横条"模板。用三层错落的有机布局，上层比下层略微偏移，形成台阶感，暗示"协同"而非"堆叠"。层间用斜向而非垂直的数据流箭头连接，体现数据在层间的流动与反馈。

### 视觉元素
- **顶层（量化优化层）**：三个圆角胶囊形模块（IQ4_XS 权重、KV q4_0、FlashAttention），并排放置但高度略有起伏，像漂浮的气泡。每个胶囊用不同浅色（薄荷绿、天蓝、淡紫）。
- **中层（运行时调度层）**：一个大的不规则有机形状（非矩形），内部包含 MoE Router Hook、prefetch_scheduler、unified_io_scheduler 三个子模块，用虚线连接表示协作关系。整体形状用柔和的波浪边界，区别于顶层的几何感。
- **底层（内核协同层）**：一个宽底座，左侧画一个圆柱体代表 NVMe SSD（GGUF 文件），右侧画一个芯片图标代表 CPU。中间用粗箭头标注 mmap + madvise 的双向数据流。
- **层间连接**：从顶层到底层，用三条不同颜色的斜向粗箭头（蓝→绿→橙），箭头上有小标签（"控制策略"、"调度决策"、"内核接口"）。

### 配色与标注
- 三层背景用极浅的渐变色（蓝/绿/橙，透明度 10%）
- 右侧悬浮三个关键数据标签：45GB→2GB RSS、+71.4% decode、0.5× KV
- 不在图内放长文字，所有细节由图注承载

## 图 2：MADV_RANDOM 按需加载机制（fig:madv_random）

### 设计意图
左右对比，用"满溢 vs 精准"的视觉张力强化 insight。左侧密密麻麻、拥挤不堪；右侧留白、精准。这种密度对比直接传达"随机预读的浪费"与"按需加载的高效"。

### 视觉元素
- **左侧（WILLNEED 默认）**：一个矩形区域，内部填满 512 个小方块（代表 512 个专家），全部填深色，密不透风。底部一个红色的"内存溢出"波浪从下方涌上来，标注 RSS=45GB。一个红色叉号悬浮在上方。
- **右侧（MADV_RANDOM）**：同样大小的矩形区域，但只有 10 个小方块填色（散布在 512 格中），其余 502 格只有淡灰色边框。大量留白。底部一个绿色的"内存充裕"平直线，标注 RSS=2GB。一个绿色对号悬浮。
- **中间分隔**：用一条虚线分割左右，中点放一个"vs"圆标。

### 配色与标注
- 左侧深红/深灰基调（压抑），右侧浅蓝/白基调（轻松）
- 底部用超大字号对比标注 "45GB → OOM" 和 "2GB → OK"
- 不画任何流程箭头，纯视觉密度对比

## 图 3：MoE Router Hook 跨层专家预测（fig:moe_router_hook）

### 设计意图
用时间轴从左到右展示 Layer N → N+1 的预测-预取流水线。用"雷达扫描"的隐喻——Layer N 的 Router 输出像雷达扫描一样"照亮"下一层需要的专家。

### 视觉元素
- **左侧（Layer N）**：一个矩形代表当前层计算，内部有一个小的 Router 图标（齿轮或决策树），输出一束扇形光线（雷达扫描状）射向右下方。
- **中间（预测路径）**：扇形光线中标注 "expert IDs [e1..e10]"，用虚线箭头表示预测。光线经过一个"缓存"图标（数据库柱体），表示 expert IDs 被缓存。
- **右侧（Layer N+1）**：一个 3D 合并张量的截面图——512 格的网格中，被光线照到的 10 格高亮发光，其余暗淡。标注"已预取的专家在 page cache 中"。
- **底部**：一条时间轴，标注 "Layer N 计算完成 → 预测 → 预取 → Layer N+1 无 page fault"。

### 配色与标注
- 发光的高亮专家用亮黄/亮橙（雷达照射感）
- 其余暗淡专家用深灰
- 雷达光线用半透明渐变扇形

## 图 4：StreamingLLM KV Eviction（fig:kv_eviction）

### 设计意图
用"传送带"隐喻。KV token 在一条传送带上从左向右移动，左端有一个固定的"sink 锁定区"（用锁链图标），右端是滑动的"window 区"（用滑轨图标），中间的 token 被一个"驱逐机械臂"抓走丢弃。

### 视觉元素
- **传送带**：一条水平的粗带子，上面有若干小方块代表 KV token，从左到右排列。
- **左端（sink 区）**：前 4 个方块用锁链图标包裹，标注"attention sink（永久保留）"。锁链用金色。
- **中间（驱逐区）**：若干灰色方块，被一个机械臂（从上方伸下）夹起并向下丢入一个"回收站"图标（垃圾桶或向下箭头）。标注"evicted → 释放给权重缓存"。
- **右端（window 区）**：最近 W 个方块用滑轨图标标注"sliding window"，方块在滑轨上向左滑动。
- **底部对比**：两条内存占用条，上条标注 O(context_length) 满格，下条标注 O(sink+window) 约 1/10 格。

### 配色与标注
- sink 区金色（珍贵固定）
- 驱逐区灰色（废弃）
- window 区蓝色（活跃滑动）
- 回收站用绿色（释放回收）

## 图 5：FlashAttention IO-aware Tiling（fig:flashattn）

### 设计意图
对比"标准 attention 的三次 I/O"与"FlashAttention 的融合单次 I/O"。用"水管粗细"隐喻内存带宽消耗——标准版三根粗水管（多次读写），FlashAttention 一根细水管（单次融合）。

### 视觉元素
- **左侧（标准 Attention）**：三个独立矩形（QK^T、softmax(S)、PV），用粗箭头串联。每个矩形上方画一根粗水管（代表 DRAM 读写），三根粗管并联，底部标注"3 次 HBM 读写"。
- **右侧（FlashAttention）**：一个融合的大矩形，内部分为若干小 tile 网格（代表 tiling）。上方只有一根细水管，底部标注"1 次 HBM 读写"。大矩形内部用循环箭头表示 tiling 的迭代融合。
- **底部对比**：用带宽条形图，标准版占用带宽高，FlashAttention 低 71%。

### 配色与标注
- 标准版用红色水管（高消耗）
- FlashAttention 用绿色水管（低消耗）
- tile 网格用浅蓝交替色

## 通用设计规范（NeurIPS/AAAI 标准）

1. **少文字多图**：每个图标/方块最多 2-3 个词，详细信息放图注
2. **不规整普通**：避免纯矩形堆叠，用圆角、斜向、错位、有机形状打破规整
3. **颜色语义**：红色=问题/baseline，绿色=优化/solution，蓝色=机制，紫色=创新，金色=固定/珍贵
4. **箭头语义**：实线=数据流，虚线=预测/控制，粗线=主要路径，渐变扇形=扫描/照射
5. **自包含图注**：图注应能独立解释图的含义，不依赖正文
6. **矢量图优先**：用 SVG/PDF 而非 PNG，保证缩放清晰
7. **字号**：图中文字 ≥ 10pt，图注 ≥ 9pt
8. **留白**：对比类图用密度差异（满 vs 空）而非文字传达信息
9. **隐喻**：善用传送带、雷达、水管、机械臂等物理隐喻，增强直观性
10. **避免模板**：不用 PowerPoint/Word 自带的规整 SmartArt，追求手绘设计感
